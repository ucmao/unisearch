## 📖 项目简介

一个基于 AI 的**多平台公开内容采集与调研工具**，支持小红书、抖音、快手、B站、微博、贴吧、知乎等主流平台。用户可以通过自然语言描述调研目标，由 AI 生成可确认、可调整的采集计划，并在任务完成后查看、筛选、对比、分析和导出结果。

项目采用 **TypeScript + Electron** 架构，提供桌面端可视化操作界面。平台登录态、任务记录和采集结果默认保存在本机；仅在用户使用 AI 规划或分析能力时，相关对话及必要数据才会发送至用户配置的模型服务。

### 🔧 技术原理

- **核心技术**：基于 [Playwright](https://playwright.dev/) 浏览器自动化框架进行数据采集。
- **运行机制**：Electron 桌面端启动本地 Fastify API 服务作为后端，通过 Web 页面控制独立的 Node 爬虫工作进程，实现多平台并行采集。
- **浏览器环境**：通过 CDP 连接 Electron 内置的 Chromium，并为不同平台维护独立登录会话，无需手动开启 Chrome 远程调试。
- **采集方式**：结合页面解析、公开接口响应和网络事件采集当前登录态可见的公开内容。
- **AI 调研**：支持自然语言任务规划、分析目标调整、采集结果总结，以及舆情和竞品对比等场景。

- **AI 任务规划**：根据自然语言目标选择平台、关键词、采集深度和分析维度，执行前可确认和调整计划。

## 🚀 快速开始

### 📋 前置依赖

1. **Node.js**：请前往官网下载安装：
   - **下载地址**：https://nodejs.org/en/download/
   - **版本要求**：>= 20.0.0（推荐使用 Node.js 20 LTS 或更高版本）

项目通过 CDP 连接 Electron 内置的 Chromium，无需安装 Chrome，也无需手动开启浏览器远程调试。

### 📦 安装依赖

在项目根目录下执行以下命令，分别安装主程序和 WebUI 的 Node 依赖包：

```shell
npm install
npm --prefix webui install
```

### 🖥️ 启动桌面端应用（Electron 开发模式）

执行以下命令，会自动编译 TypeScript 代码（通过 `tsup`）并启动 Electron 可视化界面：

```shell
npm run electron:dev
```

首次启动平台任务时，请根据界面提示使用二维码或 Cookie 完成登录。登录或验证码处理阶段建议关闭“无头浏览器”；登录成功后，平台会复用保存在本机的登录态。

### 🤖 配置 AI 模型（可选）

采集和结果查看功能可以独立运行；如需使用自然语言任务规划、对话和结果分析，请在应用的“设置 → 模型服务”中配置 API Key。

目前支持：

- MiniMax
- DeepSeek
- 自定义 OpenAI 兼容接口

API Key 使用 Electron 安全存储能力保存在本机。调用 AI 功能时，请同时遵守所选模型服务商的数据使用和隐私政策。

## 🛠️ WebUI 可视化开发与调试

项目支持前后端分离调试。由于采集任务依赖 Electron 内置 Chromium，开发时需要同时运行 Electron 后端和 Vite 前端。

### 1. 启动 Electron 后端

在项目根目录执行：

```shell
npm run electron:dev
```

Electron 会启动本地 Fastify API 服务，默认优先使用 `http://127.0.0.1:8080`；如果端口被占用，会自动选择其他可用端口。

### 2. 启动前端 Vite 开发服务器

新建一个终端窗口，进入 `webui` 目录并启动前端：

```shell
cd webui
npm run dev
```

启动成功后，在浏览器访问 `http://localhost:5173/` 即可。

> **注意：**Vite 开发服务器默认将 `/api` 请求代理到 `http://localhost:8080`。调试前请确保 8080 端口未被其他程序占用，否则需要同步调整 `webui/vite.config.ts` 中的代理地址。

### 3. 构建静态生产资源

若需要将前端代码打包，并由 Fastify 服务统一提供访问，可执行：

```shell
npm run webui:build      # 产物输出到 api/webui/
```

构建完成后，重新启动 Electron 应用即可加载最新的 WebUI：

```shell
npm run electron:dev
```
